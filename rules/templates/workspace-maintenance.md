# Workspace Structure Maintenance

## Required Updates

When making changes to the workspace project layout, you MUST update:

1. **`.amazonq/rules/project-organization.md`** - Update project descriptions

## Enforcement

Any workspace structural changes must be immediately followed by documentation updates.

## Examples of Structural Changes

- Adding new projects
- Removing or archiving projects
- Renaming project directories
- Changing project purposes

## Verification

After updates, verify that:
- Project structure diagram matches actual structure
- All projects are documented with clear purposes
- Quick reference guide includes all current projects
