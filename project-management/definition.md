# Project Management Standards

## Purpose

This domain defines standards for managing software development projects, including roadmap documentation, phase-based development, and project organization patterns.

## Scope

- Roadmap documentation and archival patterns
- Phase-based development methodology
- Project planning and tracking standards
- Documentation organization and maintenance

## Standards

- [Roadmap Documentation](roadmap-documentation.md) — Standards for creating and maintaining project roadmaps with phase-based development

## Principles

1. **Focus on Current Work** — Roadmaps emphasize current and future phases
2. **Preserve History** — Archive completed phases with full implementation details
3. **Measurable Progress** — Use concrete exit criteria and checkboxes
4. **Clear Structure** — Consistent organization across all project documentation
5. **Regular Maintenance** — Keep documentation current and accurate

This domain ensures consistent project management practices across all software development efforts.
